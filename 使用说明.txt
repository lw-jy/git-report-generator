Git Report Generator 使用说明
==============================

一、环境要求
-----------
- Windows 系统
- 安装 Node.js（下载地址：https://nodejs.org，下载 LTS 版本）
  安装时一路点「下一步」即可


二、启动步骤
-----------
1. 解压本文件夹到任意位置
2. 双击 start.bat
3. 等待几秒，浏览器会自动打开 http://localhost:3000
4. 如果浏览器没自动打开，手动输入 http://localhost:3000


三、首次配置
-----------
1. 打开页面后，点击导航栏「配置中心」
2. 根据你的代码平台填写：

   若使用 GitLab：
   ┌──────────────────────────────────────────────┐
   │ 平台         → 选 GitLab                     │
   │ GitLab 地址   → 填你们公司的 GitLab 地址      │
   │ Token        → 填你的 GitLab Personal Token  │
   │ 用户名       → 填你的 GitLab 用户名           │
   │ 仓库         → 输入 namespace/project 按回车  │
   │              可添加多个仓库                    │
   │ AI Key       → 填你的 AI API Key             │
   │ AI 地址      → 填 AI 服务地址                 │
   │ AI 模型      → 填模型名称                     │
   └──────────────────────────────────────────────┘

   若使用 GitHub：
   ┌──────────────────────────────────────────────┐
   │ 平台         → 选 GitHub                     │
   │ Token        → 你的 GitHub Personal Token    │
   │ 用户名       → 你的 GitHub 用户名             │
   │ 仓库         → 输入 owner/repo 按回车         │
   │ AI 相关      → 同上                          │
   └──────────────────────────────────────────────┘

3. 点击「保存配置」


四、生成报告
-----------
1. 点击导航栏「报告生成」
2. 选择时间范围（今天/最近3天/本周/最近两周）
3. 选择报告类型（日报/周报）
4. 点「生成报告」
5. 等待 AI 生成完成，报告会显示在页面右侧


五、自定义输出格式（可选）
-------------------------
在配置中心的「输出格式」区域，可以自定义报告的结构。
留空则使用默认格式（概述、功能开发、Bug修复、其他变更、总结计划）。
填写示例：

    请按以下格式输出：

    ### 本周完成工作
    （列出所有工作项）

    ### 本周工作总结
    （整体总结）

    ### 下周计划
    （下周工作安排）


六、提交过滤（可选）
------------------
在配置中心的「提交过滤」区域，可以按关键词排除不需要的提交。
每行一个关键词，匹配到的 commit 会被过滤掉。
常用过滤：Merge branch、WIP、chore(deps)、Bump version


七、常见问题
-----------
Q: 启动时提示"node 不是内部或外部命令"
A: 说明没有安装 Node.js，前往 https://nodejs.org 下载安装

Q: 生成报告时提示"Token 无效"
A: 检查配置中心的 Token 是否正确，是否已过期

Q: 拉取不到仓库
A: 确认 GitLab 地址是否填对，Token 是否有 read_api 权限

Q: 生成报告很慢
A: AI 生成需要时间，一般在 10-30 秒左右，属于正常情况

Q: 页面显示空白
A: 刷新一下页面，或关闭浏览器重新打开 http://localhost:3000


八、关闭程序
-----------
- 直接关掉浏览器标签页即可
- 服务在后台运行，想彻底停止请关掉命令行窗口
